## рҹ“Ң Overview

**CareConnect Egypt** is a full-stack, role-based healthcare management platform connecting **patients**, **doctors**, **hospitals**, **medical service providers**, and platform **administrators** in one system.

It handles the real, everyday friction of accessing healthcare in Egypt: finding a doctor by specialty, booking an appointment, tracking a digital insurance claim, checking whether a nearby hospital has the blood type you need вҖ” all through a modern API-driven backend and an Angular single-page frontend.

The platform is built on **Clean Architecture** principles on the backend and a fully **standalone-component Angular** frontend вҖ” no legacy NgModules, no third-party map billing, no shortcuts.

## рҹҸ—пёҸ System Architecture

```
CareConnectEgypt/
в”ңв”Җв”Җ src/
в”Ӯ   в”ңв”Җв”Җ CareConnect.Domain/           # Entities, Enums, Constants
в”Ӯ   в”ңв”Җв”Җ CareConnect.Application/      # DTOs, Interfaces, Validators, Common
в”Ӯ   в”ңв”Җв”Җ CareConnect.Infrastructure/   # EF Core, Services, Migrations, Seed
в”Ӯ   в””в”Җв”Җ CareConnect.Api/              # Controllers, Program.cs, appsettings
в”ңв”Җв”Җ tests/
в”Ӯ   в””в”Җв”Җ CareConnect.Api.IntegrationTests/
в””в”Җв”Җ careconnect-client/               # Angular app
    в””в”Җв”Җ src/app/
        в”ңв”Җв”Җ core/                     # models, services, guards, interceptors
        в”ңв”Җв”Җ features/                 # per-role pages (patient, doctor, hospital, super-admin, blood-bank, directory...)
        в”ңв”Җв”Җ layouts/
        в””в”Җв”Җ shared/                   # dialogs, shared components
```

The backend follows **Clean Architecture**: `Domain` has zero external dependencies, `Application` defines the use cases and contracts, `Infrastructure` implements persistence and external services, and `Api` wires everything together and exposes it over HTTP.

## вңЁ Features (Implemented & Working)

### рҹ”‘ Authentication & Profiles
- Registration & login with **JWT + Refresh Tokens**
- Multi-role authorization across 5 roles
- Dedicated profiles: `PatientProfile`, `DoctorProfile`, `HospitalProfile`, `MedicalServiceProviderProfile`

### рҹ©ә Specialties & Directories
- Bilingual (Arabic / English) medical specialties, managed by SuperAdmin
- Hospital вҶ” Specialty linking (`HospitalSpecialties`)
- Doctor вҶ” Hospital affiliations with request / approve / reject workflow
- Searchable directories for doctors and hospitals

### рҹ“… Appointments
- Weekly doctor availability schedules (`DoctorAvailability`)
- Doctor unavailability periods (`DoctorUnavailablePeriods`)
- Full appointment lifecycle: **Pending вҶ’ Confirmed вҶ’ Rejected вҶ’ Cancelled вҶ’ Completed вҶ’ NoShow**

### рҹӣЎпёҸ Digital Health Insurance
- Insurance requests linked to insurance companies
- Status workflow: **Pending вҶ’ UnderReview вҶ’ Approved вҶ’ Rejected вҶ’ Cancelled**
- Insurance company management from the SuperAdmin dashboard

### рҹ©ё Blood Bank
- Per-hospital blood inventory across all 8 blood types
- Patient blood requests with status workflow: **Pending вҶ’ Approved вҶ’ Rejected вҶ’ Fulfilled вҶ’ Cancelled**
- Search for blood availability at nearby hospitals

### рҹ“Қ Hospital Location & Nearby Discovery
- Each hospital stores latitude/longitude and address
- "Nearest hospital" search using the **Haversine formula** вҖ” no paid maps API
- Filter by specialty, blood availability, and appointment availability

### рҹ“Ҡ Dashboards
- Role-specific dashboards with relevant statistics for every role

### рҹҢұ Demo Data Seeder
- Idempotent seeder generating realistic demo data (hospitals, doctors, appointments, insurance, blood bank)
- Development-only, toggled via configuration

## рҹҡҖ Getting Started

### Prerequisites

- [.NET 10 SDK](https://dotnet.microsoft.com/download)
- [SQL Server](https://www.microsoft.com/en-us/sql-server) (or LocalDB)
- [Node.js](https://nodejs.org/) + npm
- [Angular CLI](https://angular.dev/tools/cli) (`npm install -g @angular/cli`)

---

### вҡҷпёҸ Backend Setup

```bash
cd src/CareConnect.Api
dotnet restore
dotnet ef database update -p ../CareConnect.Infrastructure -s .
dotnet run
```

вң… API running at: **`http://localhost:5290`**
вң… Swagger UI at: **`http://localhost:5290/swagger`**

---

### рҹҺЁ Frontend Setup

```bash
cd careconnect-client
npm install
ng serve
```

вң… App running at: **`http://localhost:4200`**
